What is OpenLRS?

OpenLRS is a secure, standards-based, standalone Learning Record Store. OpenLRS was built to fill the need for a high i/o storage mechanism for an open learning analytics environment, as displayed in the diagram on the right. Built on a scaleable architecture, using modern web technologies, OpenLRS provides the fast reads and writes necessary for a dynamic analytics environment.

Features

  • Open source
  • Developer friendly
  • Built with Spring Boot
  • xAPI support
  • IMS Caliper support in-progress
  • Support for multiple backend data stores
  • Cloud ready

Get Started

Technical Overview

OpenLRS is a Java web application built with Spring Boot. OpenLRS manages configuration using Spring Profiles. A complete listing of the supported profiles is available below.

Requirements

To build and run OpenLRS you will need, at a minimum, the following:

  • Java 7 or higher (either JDK)
  • Maven 3 or higher

Building and Running OpenLRS

To get started quickly for testing or demo purposes you can run OpenLRS with the default profile. This will start the application using an in-memory datastore (you will lose all the data when the application is stopped). 

              
              // Just build the jar file
              mvn clean package
              // this produces the openlrs.jar file in the target folder
              
            
              
              // Run in place
              mvn clean package spring-boot:run
              // this starts the application on port 8080
              
            
              
              // Run in place using the jar file
              // as an alternative to using spring-boot:run
              // you can build the jar file using mvn clean package
              // and then run
              java -jar target/openlrs.jar
              // this starts the application on port 8080

Now that you have the application running you can test it by sending some event information to the various service endpoints. Keep in mind that OpenLRS is really just a collection of web services with no user interface (if you pull up OpenLRS in a browser you'll just see a watermark image). The easiest way to run a few tests is to use a rest client like Postman. Once you have Postman installed you can import this collection of requests (instructions for importing colletions can be found here).

Branches

Master Branch

The OpenLRS master branch will always contain the most stable version of the code base. If you are just starting out with OpenLRS we recommend that you start with the master branch and, of course, if you plan to deploy OpenLRS in a pilot or production environment use the master branch.

Development Branch

The OpenLRS development branch contains that latest code changes and any experimental features that have not yet been deemed production ready. Support for IMS Caliper is currently in the development branch and relies on the IMS Caliper Java library - make sure you build and install this library with maven before you try to build the development branch.

GH Pages Branch

The OpenLRS gh-pages branch contains the HTML and supporting assets for this web page.
